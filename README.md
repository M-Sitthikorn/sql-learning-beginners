## RULE FOR SQL LANGUAGE 
1.   ความไม่แตกต่างระหว่างตัวพิมพ์เล็กและตัวพิมพ์ใหญ่ `(Case-insensitive)`
    - การใช้ตัวพิมพ์เล็กหรือใหญ่ไม่มีผลต่อการทำงานของคำสั่ง
    - ข้อยกเว้น: ข้อความที่อยู่ภายในเครื่องหมายคำพูด `''` หรือ `""` จะถือว่ามีความแตกต่าง เช่น `'USA'`, `'McDonalsd'`
2. การเขียนคำสั่ง (Clauses และ Keywords)
    - ให้เขียน SQL Clauses และ Keywords ด้วย ตัวพิมพ์ใหญ่ เพื่อความชัดเจนและมาตรฐาน
    - ตัวอย่าง: 
```sql
SELECT name, country 
FROM customers
WHERE country = 'USA';
```

3. การเขียนหลายคำสั่ง (`Multiple Queries`)
    - หากมีการเขียนมากกว่าหนึ่งคำสั่งในหน้าต่าง editor ต้องใส่เครื่องหมาย ; (semicolon) หลังแต่ละคำสั่ง
    - เครื่องหมาย ; ทำหน้าที่เหมือนเครื่องหมายจุด (.) ในประโยคภาษาอังกฤษ เพื่อบ่งบอกว่าคำสั่งสิ้นสุดแล้ว
    - ตัวอย่าง: 
```sql
SELECT * FROM employees;
SELECT * FROM departments;
```



### 1.1 DML - Data Manipulation Language

- `INSERT INTO`: ใช้สำหรับเพิ่มข้อมูลใหม่ลงในตาราง
- `UPDATE`: ใช้แก้ไขข้อมูลที่มีอยู่แล้วในตาราง (ต้องระวังการใช้คู่กับ WHERE)
- `DELETE`: ใช้ลบข้อมูลออกจากตาราง
- `ORDER BY`: ใช้เรียงลำดับข้อมูล (เช่น ASC น้อยไปมาก, DESC มากไปน้อย)
- `LIMIT`: (เฉพาะใน Postgres/MySQL) ใช้จำกัดจำนวนแถวที่ต้องการแสดงผล
- `DISTINCT`: ใช้เลือกเฉพาะข้อมูลที่ไม่ซ้ำกันออกมา

### 1.2 DDL - Data Definition Language

- `CREATE TABLE`: ใช้สร้างตารางใหม่
- `ALTER TABLE`: ใช้แก้ไขโครงสร้างตาราง เช่น เพิ่มคอลัมน์ หรือเปลี่ยนชื่อคอลัมน์
- `DROP TABLE`: ใช้ลบตารางออกจากระบบ (ข้อมูลหายทั้งหมด)
- `TRUNCATE`: ใช้ลบข้อมูลทั้งหมดในตารางแต่ยังเก็บโครงสร้างตารางไว้

### 1.3 DQL - Data Query Language 

#### The Essentials

- `SELECT`: ระบุชื่อคอลัมน์ที่ต้องการ (ใช้ * เพื่อเลือกทุกคอลัมน์)
- `FROM`: ระบุชื่อตารางหลักที่ข้อมูลเก็บอยู่
- `AS` (Aliasing): ใช้ตั้งชื่อเล่นให้คอลัมน์หรือตารางเพื่อให้เรียกง่ายหรืออ่านผลลัพธ์ง่ายขึ้น

#### Filtering
- `WHERE`: กำหนดเงื่อนไข เช่น WHERE price > 100
- `AND` / `OR` / `NOT`: ตัวดำเนินการทางตรรกะเพื่อรวมหลายเงื่อนไข
- `IN`: ตรวจสอบว่าค่าอยู่ในรายการที่กำหนดหรือไม่ เช่น IN ('USA', 'UK', 'Thailand')
- `BETWEEN`: ตรวจสอบว่าค่าอยู่ในช่วงที่กำหนดหรือไม่ เช่น BETWEEN 10 AND 50
- `LIKE` / `ILIKE`: ใช้ค้นหาข้อความตามรูปแบบ (ILIKE ใน Postgres จะไม่สนใจตัวพิมพ์เล็ก-ใหญ่) เช่น  `LIKE` 'A%' (ขึ้นต้นด้วย A)
- `IS NULL` / `IS NOT NULL`: ใช้ตรวจสอบค่าว่าง

#### Result Processing

- `ORDER BY`: เรียงลำดับข้อมูล (ASC จากน้อยไปมาก, DESC จากมากไปน้อย)
- `LIMIT`: จำกัดจำนวนแถวที่จะแสดงผล (เช่น LIMIT 10)
- `OFFSET`: ข้ามจำนวนแถวที่ระบุ (มักใช้คู่กับ LIMIT เพื่อทำระบบแบ่งหน้า/Pagination)
- `DISTINCT`: ดึงเฉพาะค่าที่ไม่ซ้ำกันออกมา

#### Aggregation
- `GROUP BY`: จัดกลุ่มแถวที่มีค่าเหมือนกัน เพื่อใช้กับฟังก์ชันคำนวณ (เช่น นับจำนวนแยกตามรายชื่อประเทศ)
- `HAVING`: ใช้กรองข้อมูล หลัง การจัดกลุ่ม (คล้าย WHERE แต่ใช้กับผลลัพธ์จากการ GROUP BY)

### Writing Order

- `SELECT` (เลือกอะไร?)
- `FROM` (จากไหน?)
- `JOIN` (เชื่อมกับอะไร?)
- `WHERE` (กรองแถว?)
- `GROUP BY` (จัดกลุ่ม?)
- `HAVING` (กรองกลุ่ม?)
- `ORDER BY` (เรียงลำดับ?)
- `LIMIT / OFFSET` (จะเอาไปแสดงกี่แถว)


```sql 
SELECT 
    c.id,
    c.first_name,
    c.order_id,
    o.sales
FROM customers AS c
INNER JOIN orders AS o
ON c.id = o.customer_id

```
